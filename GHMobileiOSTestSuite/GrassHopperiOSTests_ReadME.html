GrassHopper Mobile iOS Test Documentation 


Summary: 

This Document walks you through what tool is being used for GrassHopper - Mobile iOS Automation, how to install and set it up, structure of the code in detail, how to run tests on local machine. 


Pre-Requisite: 

    

Refer to these pages for detailed instructions on how to install Appium and other ios dependencies. 


https://confluence.ops.expertcity.com/display/GH/Robot+Framework+-+Installation+Guidelines


   Install External Libraries: 


       To Run tests inside this repo, install the following external libraries:

1. AppiumLibrary — pip install robotframework-appiumlibrary



  Run Appium on your local machine:

         Open terminal and say ‘Appium’ to start appium server. 



Directory Structure:



GHMobileiOSTestSuite is the root directory for this code. 


DEPENDENCIES.txt:


This file has all the Libraries, Resource files that are needed to Run the GrassHopper Mobile Tests. 


Environment Directory:


Contains .txt files that represent specific environment configurations where the tests need to run. 


For example, if you want to run tests against a real device, save all of device ID and other details into this file.

You can create more files to represent each device. 


Resources Directory:


This directory contains 3 files,


Keywords.txt —> GrassHopper specific keywords that we write go in here. 

Locators.txt —> All the Locators on Mobile iOS App need to be in this file. 

Variables.txt —> Variables that are constant across various environments need to be in this file. For example, plan names, Pricings and others that are common for GH application can be added to this file. 


TestResults Directory:


This directory is to save the test results after execution. When tests fails, screenshots are saved to this directory as well. 


TestSuite Directory:


This directory contains the test cases. Test Cases could be grouped by features, modules, functionality or any other way. Depending on how test cases are grouped, it could refer to different Resources files if needed. 


Tagging Test cases:


We could use ‘[tags]’ notation to add tags to your test cases. For ex, say your test case is testing inbox related features, tag them ‘inbox’ and when you want to run all test cases related to inbox, use this tag. 


Running Tests:


There are many ways to run test cases from command line. Refer to this page for details. 

http://robotframework.org/robotframework/latest/RobotFrameworkUserGuide.html#all-command-line-options


open terminal and cd into the root directory (GrassHopperPortalAutomation)

$ pybot --variable environment:RealDevice1 --outputdir TestResults TestSuite


-- variable is used to pass in variables to the tests from command line. In this case, we are passing the environment name


-- outputdir is used to specify the location of the result files, in this case we are saying save under TestResults directory


TestSuite is the location of the test case files


In this case, all test cases inside TestSuite directory would be run.
